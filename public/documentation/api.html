API reference

CrossSwap exposes a small REST proxy under /api/*. The storefront uses these endpoints; you can call them from your own clients too.

On this page: Overview · Exchange · Support · Chat · Limit orders · Recurring · Errors · Rate limits

Overview

  • Base URL: https://yourdomain.tld/api
  • Content type: application/json for both requests and responses.
  • Auth: anonymous for exchange / support / chat; the limit-order and recurring endpoints require a signed-in session (auth:web).
  • All amounts are floats; currency tickers are lowercase.
  • Rate-limited via Laravel's throttle middleware — see Rate limits.

The ChangeNOW partner key the REST proxy uses is configured from the admin panel:

Exchange endpoints

Public, anonymous. Rate-limited to throttle:api (60 req/min by default).

GET/api/exchange/currencies

Returns the supported currency list. Cached for five minutes.

Query parameters

flow
Optional. standard (default) or fixed-rate.

Example curl

curl https://yourdomain.tld/api/exchange/currencies?flow=standard

Example response

[
  {
    "ticker": "btc",
    "name": "Bitcoin",
    "network": "btc",
    "image": "https://.../btc.svg",
    "featured": true,
    "has_external_id": false
  }
]

GET/api/exchange/min-amount

Minimum send amount for a pair. Cached for one minute.

Query parameters

from
Source ticker, e.g. btc.
to
Destination ticker.
from_network
Optional network override.
to_network
Optional network override.
flow
standard or fixed-rate.

Example curl

curl "https://yourdomain.tld/api/exchange/min-amount?from=btc&to=eth&flow=standard"

Example response

{ "min_amount": 0.0005 }

GET/api/exchange/estimate

Returns the expected receive amount and live exchange rate.

Query parameters

from
Source ticker.
to
Destination ticker.
amount
Send amount (float).
flow
standard or fixed-rate.

Example curl

curl "https://yourdomain.tld/api/exchange/estimate?from=btc&to=eth&amount=0.05&flow=fixed-rate"

Example response

{
  "from": "btc",
  "to": "eth",
  "amount_send": 0.05,
  "amount_receive": 0.834120000000,
  "rate_id": "f1a2...",
  "valid_until": "2026-06-14T12:31:00+00:00",
  "flow": "fixed-rate",
  "warning": null
}

POST/api/exchange/create

Creates a swap and stores it locally. Returns a CrossSwap transaction row.

Request body

{
  "from": "btc",
  "to": "eth",
  "amount": 0.05,
  "address": "0x...",
  "refund_address": "bc1q...",
  "rate_id": "f1a2...",
  "flow": "fixed-rate",
  "email": "user@example.com"
}

Example curl

curl -X POST https://yourdomain.tld/api/exchange/create \
  -H "Content-Type: application/json" \
  -d '{"from":"btc","to":"eth","amount":0.05,"address":"0x...","flow":"standard"}'

Example response

{
  "id": 42,
  "provider_id": "abc123",
  "status": "waiting",
  "payin_address": "bc1q...",
  "amount_send": 0.05,
  "amount_receive": 0.8341,
  "valid_until": "2026-06-14T12:31:00+00:00"
}

GET/api/exchange/status/{id}

Returns the latest status for a transaction. {id} is the provider_id returned by create.

Example curl

curl https://yourdomain.tld/api/exchange/status/abc123

Example response

{
  "provider_id": "abc123",
  "status": "confirming",
  "payin_hash": "txhash...",
  "payout_hash": null,
  "amount_receive": 0.834120
}

POST/api/exchange/validate

Checks whether an address looks valid for the given currency.

Request body

{ "currency": "btc", "address": "bc1q..." }

Example curl

curl -X POST https://yourdomain.tld/api/exchange/validate \
  -H "Content-Type: application/json" \
  -d '{"currency":"btc","address":"bc1q..."}'

Example response

{ "valid": true }

Support

POST/api/support/ticket

Opens a support ticket. Used by the contact form.

Request body

{
  "email": "user@example.com",
  "subject": "Swap is stuck",
  "message": "...",
  "tx_id": "abc123"
}

Example curl

curl -X POST https://yourdomain.tld/api/support/ticket \
  -H "Content-Type: application/json" \
  -d '{"email":"user@example.com","subject":"Swap is stuck","message":"My swap..."}'

Example response

{ "id": 17, "status": "open" }

Chat

POST/api/chat/message

Send a message to the AI assistant and receive a streamed reply. Throttled to 20 requests per minute per IP.

Request body

{
  "history": [
    {"role": "user", "content": "How long does a BTC -> ETH swap take?"}
  ]
}

Example curl

curl -X POST https://yourdomain.tld/api/chat/message \
  -H "Content-Type: application/json" \
  -d '{"history":[{"role":"user","content":"hi"}]}'

Example response

{
  "reply": "Most BTC to ETH swaps complete in 15-30 minutes...",
  "provider": "anthropic",
  "model": "claude-haiku-4-5"
}

Limit orders

Customer-only. Requires a signed-in session via auth:web; the request must include the CSRF token from <meta name="csrf-token">. Rate-limited to throttle:api.

GET/api/limit-orders

Lists the signed-in customer's open and historical limit orders.

Example response

[
  {
    "id": 12,
    "from": "btc",
    "to": "usdt",
    "amount": 0.01,
    "target_rate": 75000,
    "status": "open",
    "created_at": "2026-06-10T11:02:00+00:00"
  }
]

POST/api/limit-orders

Creates a new limit order — fires when the live rate crosses target_rate.

Request body

{
  "from": "btc",
  "to": "usdt",
  "amount": 0.01,
  "target_rate": 75000,
  "address": "0x...",
  "expires_at": "2026-07-01T00:00:00+00:00"
}

Example response

{ "id": 12, "status": "open" }

POST/api/limit-orders/{id}/cancel

Cancels a limit order before it fires. {id} is numeric.

Example response

{ "id": 12, "status": "cancelled" }

Recurring (DCA) schedules

Customer-only. Same auth + throttle rules as limit orders.

GET/api/recurring

Lists the signed-in customer's recurring (DCA) schedules.

Example response

[
  {
    "id": 4,
    "from": "usdt",
    "to": "btc",
    "amount": 50,
    "interval": "weekly",
    "next_run_at": "2026-06-21T10:00:00+00:00",
    "status": "active"
  }
]

POST/api/recurring

Creates a recurring schedule.

Request body

{
  "from": "usdt",
  "to": "btc",
  "amount": 50,
  "interval": "weekly",
  "address": "bc1q...",
  "starts_at": "2026-06-14T10:00:00+00:00"
}

Example response

{ "id": 4, "status": "active" }

POST/api/recurring/{id}/cancel

Cancels a recurring schedule. Past runs are preserved.

Example response

{ "id": 4, "status": "cancelled" }

POST/api/recurring/{id}/pause

Pauses the schedule. Future runs are skipped until resumed.

Example response

{ "id": 4, "status": "paused" }

POST/api/recurring/{id}/resume

Resumes a paused schedule. The next_run_at is recalculated from the original interval.

Example response

{ "id": 4, "status": "active" }

Error responses

All endpoints return a JSON error body on failure:

{ "error": "min_amount", "message": "Amount is below the minimum for this pair." }

HTTP status codes follow standard conventions: 400 for validation, 422 for upstream rejection, 429 when rate-limited, 502 when the upstream API is unreachable.

Rate limits

The default throttle:api bucket is 60 requests per minute per IP. The chat endpoint uses a stricter 20 requests per minute per IP bucket because LLM calls are expensive.

Tune both in app/Providers/AppServiceProvider.php via the RateLimiter::for('api', ...) definition, or in routes/api.php at the middleware level.
