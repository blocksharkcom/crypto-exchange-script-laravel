Customization

Brand colours, languages, logos, and email templates — this page covers the small handful of files you will edit when you want to go beyond the admin panel.

On this page: Branding via admin · File structure · New language · Brand tokens · Logo · Favicon & OG · Email templates · Components

Branding via the admin panel

For 90% of buyers, the admin panel covers everything — no code edits or rebuilds required. Open Admin → Settings → Brand and you'll find:

Logo
SVG or PNG, uploaded through LogoUploader. Saved to storage/app/public/brand/ and served via the public symlink.
Logo (dark)
Optional alternate that the storefront uses on light backgrounds.
Name
Public-facing brand name. Header, footer, page titles, and emails all read this value.
Tagline
Optional. Appears under the hero on the homepage.
Social links
Twitter / X, Telegram, Discord, Reddit — left blank, the corresponding icon is hidden.
Legal URLs
Terms, Privacy, AML — these power the address-entry consent links and the footer.

Changes take effect immediately — settings are cached for 10 minutes but the admin form busts the cache on save.

Tip. The LogoUploader component validates both file type and dimensions client-side. Keep the source SVG — you'll want it later for the favicon and OG card too.

File structure

The bits of the codebase you're most likely to touch:

app/
Laravel application code — controllers, models, services. Read-only for most users; out of scope for cosmetic edits.
resources/css/app.css
Tailwind 4 entry plus the @theme block with brand colours.
resources/js/Pages/
One Inertia page per route — Home.tsx, Support.tsx, etc.
resources/js/components/
Re-usable React components grouped by feature.
resources/views/emails/
Blade templates for outbound mail.
lang/<locale>/site.php
All storefront strings, including FAQ, hero, footer, errors.
lang/<locale>/emails.php
Strings used inside email templates.
config/swapforge.php
App-level config: language list, thresholds, social links.
public/favicon.svg / public/og-image.svg
Static brand assets served as-is.

Adding a new language

CrossSwap ships with English, Spanish, German, and French translations. Adding another locale takes three steps:

  1. Edit config/swapforge.php and append the locale code to the languages array, e.g. 'it' for Italian.
  2. Copy lang/en/site.php to lang/<code>/site.php and translate the values. Every key must exist — missing keys fall back to English.
  3. Optionally translate lang/<code>/emails.php and lang/<code>/help_articles.php.

The file is a flat PHP array. A truncated sample:

<?php
return [
    'hero' => [
        'title_a'  => 'Scambia qualsiasi crypto',
        'title_b'  => 'all\'istante',
        'subtitle' => 'Scambia Bitcoin e oltre 900 token...',
    ],
    'nav' => [
        'home'     => 'Home',
        'help'     => 'Aiuto',
        'faq'      => 'FAQ',
        'docs'     => 'Documenti',
        'language' => 'Lingua',
        'track'    => 'Traccia ordine',
    ],
    // ... continue for every top-level key in lang/en/site.php
];

Clear caches after the change:

php artisan cache:clear
php artisan view:clear

Tailwind v4 brand tokens

CrossSwap uses Tailwind CSS v4 with the new @theme block. Every brand colour, ink shade, font, and accent lives in resources/css/app.css:

@theme {
    --font-sans: 'Inter', system-ui, -apple-system, 'Segoe UI', Roboto, 'Helvetica Neue', Arial, sans-serif;
    --font-mono: ui-monospace, SFMono-Regular, 'JetBrains Mono', Menlo, Consolas, monospace;

    /* Brand — pastel lime that matches reference */
    --color-brand-50:  #f3fce0;
    --color-brand-100: #e6f9b3;
    --color-brand-200: #d3f483;
    --color-brand-300: #bff15a;   /* primary CTA */
    --color-brand-400: #b1ea3e;
    --color-brand-500: #97d722;
    --color-brand-600: #79b416;
    --color-brand-700: #56850e;
    --color-brand-ink: #0a0a0c;

    /* Cyan accent — used in gradient headline */
    --color-cy-300: #6ce7d2;
    --color-cy-400: #4cd1c1;

    /* Ink scale, calibrated for true near-black */
    --color-ink-50:   #0a0c10;
    --color-ink-100:  #0e1014;
    --color-ink-200:  #14171c;
    --color-ink-300:  #1a1d23;
    --color-ink-400:  #232730;
    --color-ink-500:  #353a47;
    --color-ink-600:  #4a5160;
    --color-ink-700:  #6b7281;
    --color-ink-800:  #a8afbd;
    --color-ink-900:  #ebeef3;

    --color-progress:       #6f6df4;
    --color-progress-track: rgba(110, 109, 244, 0.16);
}

Edit the hex values then rebuild the assets:

npm install
npm run build

Tokens map directly to Tailwind utilities: --color-brand-300 becomes bg-brand-300, text-brand-300, etc. The two surface scales (--surface-* and --text-*) are defined inside @layer base further down in the same file — tweak them to flip dark / light defaults.

No rebuild needed for the primary CTA. The admin panel's Settings → Theme → Primary colour field overrides --color-brand-300 at runtime via an inline style block. Use it for one-off tweaks; reach for the @theme block when you want the change baked into the production bundle.

Replacing the logo

For the in-app logo (header and footer), upload from Admin → Settings → Brand → Logo; the file is saved to storage/app/public/brand/ and a public symlink is created when you run php artisan storage:link. Both light and dark variants can be uploaded so the storefront swaps automatically.

Custom favicon & Open Graph image

Two static assets ship in public/ and are referenced from every layout. Replace either file in place — no rebuild necessary because they're served directly:

public/favicon.svg
The browser tab icon. Keep it square; 32×32 is the design target. SVG renders sharp at every size and supports automatic dark-mode flipping via prefers-color-scheme.
public/favicon.ico
Fallback for older browsers. Optional — modern browsers always prefer the SVG.
public/apple-touch-icon.png
180×180 PNG used by iOS when a visitor adds the site to their home screen.
public/og-image.png
The 1200×630 Open Graph card used by Twitter, Discord, Slack, and Facebook link previews. Keep important content centred in a 1200×600 safe area.
public/manifest.json
PWA manifest. Update name, short_name, and the icons[] paths if you rename the icon files.

After replacing icons, hard-refresh (Cmd+Shift+R / Ctrl+F5) — browsers cache favicons very aggressively.

Reminder. Don't rely on view:clear for static assets — these files are served raw by your web server, never by Laravel, so the framework's cache commands have no effect on them.

Customising email templates

All transactional emails live in resources/views/emails/:

emails/swap_receipt.blade.php
Sent to the customer when a swap finishes.
emails/admin_swap_failed.blade.php
Sent to admins when a swap fails or gets flagged stuck.
emails/ticket_reply.blade.php
Sent when an admin replies to a support ticket.
emails/recurring_run.blade.php
Sent after a recurring (DCA) schedule fires.
emails/layout.blade.php
Shared shell — logo, footer, brand-coloured CTA button.

The templates use standard Blade. Strings come from lang/<locale>/emails.php so translations stay in sync. After editing, clear the view cache:

php artisan view:clear

Preview an email locally without sending it:

php artisan tinker --execute="\
  echo (new \\App\\Mail\\SwapReceipt(\\App\\Models\\Transaction::first()))->render();"

Deeper UI edits

For component-level changes, the React source lives under resources/js/:

  • resources/js/Pages/Home.tsx — the main storefront page.
  • resources/js/Pages/Support.tsx — the contact form.
  • resources/js/Pages/Track.tsx — order tracking.
  • resources/js/components/exchange/ — the swap widget components (currency picker, amount input, address entry).
  • resources/js/layouts/PublicLayout.tsx — header & footer.

After editing JS or CSS, rebuild the assets:

npm install
npm run build

During development, use the Vite dev server for hot reload:

npm run dev
Note. The admin panel pages live under resources/js/Pages/Admin/ and follow the same convention. They use the same brand tokens, so colour changes propagate automatically.
