Frequently asked questions

The customer-facing questions visible on the storefront, plus a handful of operator-facing ones about updates, hosting, refunds, and KYC.

Jump to: Customer FAQ · Operator FAQ

Customer FAQ

These six questions are pulled verbatim from lang/en/site.php → faq.* and are the ones your visitors see on the public site.

Do I need an account?

No. There's no sign-up or wallet connection. Just enter your receive address and send your deposit.

How long does an exchange take?

Most swaps complete in 5 to 30 minutes. Chains with slower confirmations may take longer.

What's the difference between floating and fixed rate?

Floating uses the market rate at the moment your deposit confirms. Fixed locks the rate at the moment you start the swap.

What if my swap takes too long?

Open a ticket from the Support page with your transaction ID. We'll trace it on-chain and resolve quickly.

Are there limits?

Each pair has its own minimum. Maximum depends on available liquidity and is shown live during the swap.

How are fees calculated?

Fees are baked into the displayed exchange rate. No hidden surprises before you confirm.

Operator FAQ

Questions buyers ask the most after purchasing the script.

How do I update CrossSwap to a new version?

  1. Back up .env, storage/, and your database.
  2. Re-download the latest archive from your Envato purchases page.
  3. Overwrite the files except .env and storage/.
  4. Run the upgrade routine: 
    composer install --no-dev --optimize-autoloader
php artisan migrate --force
php artisan view:clear
php artisan config:clear
php artisan horizon:terminate
  5. Hard-refresh the storefront and confirm Horizon shows new workers.

The migration runner is idempotent — running it twice will not re-apply already-applied migrations.

How do refunds work?

CrossSwap itself never holds customer funds, so refunds are issued by ChangeNOW. When a customer needs one (wrong network, wrong address, expired rate, etc.):

  1. Open the transaction in Admin → Transactions and copy the provider_id.
  2. Log into the ChangeNOW partner dashboard and use the “Request refund” flow on that transaction.
  3. ChangeNOW will ask the customer for a refund address (different from the receive address). The address-entry step on the public storefront already collects a refund_address when the customer enables fixed-rate, which speeds this up.
  4. Once ChangeNOW processes the refund, the CrossSwap transaction status updates automatically through the next polling pass.

Is CrossSwap KYC'd?

No. CrossSwap is a thin storefront on top of ChangeNOW, which is a non-custodial swap provider that doesn't require KYC for the vast majority of swap pairs and amounts. ChangeNOW may individually flag suspicious transactions and ask the sender to verify identity per their AML policy, but you as the operator are not handling any KYC paperwork yourself.

That said: depending on your jurisdiction you may still need to publish a privacy policy and an AML notice. Set the URLs under Settings → Legal.

Can I deploy CrossSwap on shared hosting?

Yes, with two caveats:

  • You need PHP 8.2+ with the extensions listed under Requirements. Most modern cPanel / Plesk / CyberPanel hosts offer this.
  • You need either a long-lived process (rare on shared hosting) or a cron job that runs every minute. Without it, status polling will not work and swaps stall at “Waiting”.

For a smoother experience on a budget, a $5-$10/month VPS with SSH access is much easier than wrestling with shared-hosting quirks.

Where can I see my licence terms?

The bundled LICENSE.md covers both the Regular and Extended Envato licences and the conditions under which you may rebrand or sell access.

Is the source code obfuscated?

No. The PHP source ships unencoded — strict typing, full namespaces, fully readable. The compiled JS bundle is minified by Vite as is normal for production builds; the un-minified source under resources/js/ is also included so you can rebuild.

Where do I report bugs?

Use the comments or item-support tab on the CrossSwap CodeCanyon listing. Include the version, PHP version, and a snippet from storage/logs/laravel.log.

More operator questions

The questions buyers ask once they're past the first install. See Troubleshooting for symptom-driven fixes; this section answers “how do I…”.

How do I update CrossSwap?

The short version: back up .env and the database, drop the new files in (preserve .env and storage/), then run:

php artisan migrate --force
php artisan optimize:clear
php artisan horizon:terminate

Only run npm run build if the changelog mentions asset changes — the archive ships a compiled bundle. Full step-by-step is on the Troubleshooting page.

Can I add a new language?

Yes — CrossSwap is fully translatable. Copy lang/en/site.php to lang/<code>/site.php, translate the values, add the locale code to config/swapforge.php → languages, then run php artisan optimize:clear. The detailed walkthrough lives on the Configuration page.

Where do I edit the homepage copy?

Two places, depending on what you want to change:

  • Marketing sections (Highlights, Reviews, Stats, How it works) — /admin/content with per-locale tabs. No code required.
  • Hero headline / tagline / nav copy — lang/<locale>/site.php under the hero.* and nav.* keys. After editing, run php artisan optimize:clear.

How do I change theme colours?

For the primary CTA only, use Admin → Settings → Theme → Primary colour — no rebuild needed. For the full palette (brand 50–700, ink scale, accents), edit the @theme block in resources/css/app.css and run npm run build. Both flows are documented on the Customization page.

Where are my customer emails?

Transactional emails (receipts, ticket replies, recurring runs) are sent via the SMTP credentials in Admin → Settings → SMTP. The Send test email button on that page round-trips a real message so you can confirm delivery. If mail isn't arriving, check storage/logs/laravel.log for transport errors and verify your MAIL_FROM_ADDRESS passes SPF/DKIM for the sending domain.

How do I disable the AI chat widget?

Open Admin → Settings → AI and set Provider to none, then hit Save. The floating chat bubble is removed from the public storefront immediately — the storefront checks the provider on render and skips the widget entirely when disabled. To re-enable later, set the provider back to openai or anthropic and paste the key.

Why isn't my SSL working?

The most common causes:

  • APP_URL in .env still starts with http:// — switch to https:// and run php artisan config:clear.
  • Behind Cloudflare or a load balancer — configure the TrustProxies middleware to honour X-Forwarded-Proto, otherwise Laravel keeps generating http:// URLs internally.
  • Mixed content from a hard-coded http:// resource — check the browser console; the offending URL will be flagged.
  • Certificate not yet issued — for Let's Encrypt, give Certbot a few seconds; for Cloudflare, ensure the SSL mode is at least Full, not Flexible with self-signed origin.

How do I migrate from SQLite to MySQL?

SQLite is great for installing fast, but you'll want MySQL or PostgreSQL once traffic picks up. The clean approach:

  1. Create the empty MySQL database and a user with full privileges on it.
  2. Export the SQLite data: 
    php artisan db:dump --database=sqlite > backup.sql
# or use the bundled dumper:
sqlite3 database/database.sqlite .dump > backup.sql
  3. Update .env: 
    DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=crossswap
DB_USERNAME=crossswap
DB_PASSWORD=...
  4. Run migrations against the empty MySQL database, then import the data: 
    php artisan migrate --force
mysql -u crossswap -p crossswap < backup.sql
  5. Clear caches and restart Horizon: 
    php artisan optimize:clear
php artisan horizon:terminate
Back up first. Always copy database/database.sqlite and .env aside before touching either. If anything goes wrong, restoring is a one-line move-back.
