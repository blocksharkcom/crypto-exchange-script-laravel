Troubleshooting

Quick recipes for the issues operators hit most often. If your problem isn't covered here, check storage/logs/laravel.log first — it almost always points at the root cause.

Jump to: Updating CrossSwap · Common errors · Stuck transactions · ChangeNOW key · Queue worker · Mail · Migrations · Permissions · HTTPS mixed content · Installer loop · Admin login

Updating CrossSwap

CrossSwap updates are delivered as a fresh archive on your Envato “Downloads” page. Every update is additive — you drop the new files over the old ones and run migrations.

Always back up .env and the database before upgrading. If anything goes wrong, restoring is a one-line move-back; without a backup, your only option is starting from scratch.

Step-by-step

  1. Back up — copy .env and your database aside: 
    cp .env .env.backup-$(date +%F)

# SQLite
cp database/database.sqlite database/database.sqlite.backup-$(date +%F)

# MySQL
mysqldump -u crossswap -p crossswap > backup-$(date +%F).sql

# PostgreSQL
pg_dump crossswap > backup-$(date +%F).sql
  2. Put the site into maintenance mode so visitors see a friendly notice while you upload: 
    php artisan down --refresh=30 --retry=60
  3. Drop the new files in place — preserve .env and the entire storage/ directory. Everything else can be overwritten safely. (Most operators use rsync --exclude or unzip with overwrite.)
  4. Run migrations — the migration runner is idempotent, so it's safe to run even when there's nothing new: 
    php artisan migrate --force
  5. Rebuild assets only if the changelog says so — the archive ships a compiled public/build/ bundle. Skip this step otherwise. 
    npm install && npm run build
  6. Clear caches so the new config, routes, and views are picked up: 
    php artisan optimize:clear
  7. Restart the queue worker so it loads the new code: 
    php artisan horizon:terminate
# or, if you use queue:work directly:
sudo systemctl restart crossswap-horizon
  8. Bring the site back up: 
    php artisan up
  9. Verify — sign into the admin panel, confirm the dashboard renders, and place a tiny test swap to confirm the upstream API still works.

Common errors & copy-paste fixes

Symptoms first, fixes second. Most of these resolve in under a minute.

419 PAGE EXPIRED

Symptom: form submissions (sign-in, ticket reply, admin save) bounce back to a “Page Expired” error page.

Cause: CSRF token mismatch — usually a stale session, a session-driver misconfiguration, or a domain mismatch.

# 1. Clear session storage:
php artisan session:flush      # file/database driver
redis-cli flushdb              # if SESSION_DRIVER=redis

# 2. Rebuild config + view cache:
php artisan optimize:clear

# 3. Confirm SESSION_DOMAIN matches APP_URL:
#    SESSION_DOMAIN=.yourdomain.tld   # leading dot for subdomains
#    APP_URL=https://yourdomain.tld

# 4. Hard-refresh the browser (Cmd+Shift+R / Ctrl+F5).

500 server error

Symptom: every request returns a blank or branded “500” page.

The exception is always in storage/logs/laravel.log:

tail -n 100 storage/logs/laravel.log

If that log is empty, the failure happened before Laravel could boot. Check the web server log:

tail -n 100 /var/log/nginx/error.log
# or
tail -n 100 /var/log/apache2/error.log

Most common causes in order of frequency: APP_KEY blank in .env (run php artisan key:generate), missing PHP extension, wrong DB credentials, and read-only storage/.

Permission denied on storage/

Symptom: “The stream or file could not be opened” or “failed to open stream: Permission denied”.

# Replace www-data with your actual PHP-FPM user if different:
sudo chown -R www-data:www-data storage bootstrap/cache
sudo find storage bootstrap/cache -type d -exec chmod 775 {} \;
sudo find storage bootstrap/cache -type f -exec chmod 664 {} \;
php artisan optimize:clear

ChangeNOW returned 401

Symptom: every swap-create or estimate fails with “Service is temporarily unavailable”; storage/logs/laravel.log shows “Authentication failed” or a 401 from api.changenow.io.

# 1. Confirm the key is set in the admin panel:
#    /admin/settings → API → ChangeNOW API key.

# 2. Re-copy it from https://changenow.io/affiliate
#    (no leading or trailing whitespace).

# 3. Verify the key against the live API:
curl -H "x-changenow-api-key: YOUR-KEY" \
  https://api.changenow.io/v2/exchange/currencies | head

# 4. If the partner account is brand new, allow 5-10 minutes
#    for the key to activate on ChangeNOW's side.

HTTPS mixed content

Symptom: the browser console shows “Mixed Content: The page was loaded over HTTPS, but requested an insecure resource” — usually images, fonts, or API calls over plain HTTP.

# 1. Set APP_URL to your https domain in .env:
APP_URL=https://yourdomain.tld

# 2. Force https in AppServiceProvider::boot()
#    (already wired when APP_ENV=production).

# 3. If behind Cloudflare / a load balancer,
#    configure TrustProxies to honour X-Forwarded-Proto.

# 4. Clear caches:
php artisan optimize:clear

Stuck transactions

A swap that sits in waiting or confirming longer than the configured thresholds is automatically marked stuck_flagged = true and surfaced in Admin → Stuck swaps. To investigate:

  1. Open the transaction in the admin panel.
  2. Compare the payin_hash against the source blockchain explorer.
  3. If the deposit never reached the payin address, the customer probably sent on the wrong network — issue a manual refund through the ChangeNOW partner dashboard.
  4. If the deposit arrived but the swap isn't progressing, contact ChangeNOW support with the provider_id.

Tune the thresholds in config/swapforge.php → tx_stuck:

'tx_stuck' => [
    'waiting_minutes'    => 180,
    'confirming_minutes' => 360,
],

ChangeNOW API key not configured (or rejected)

Symptoms: every estimate or create request fails with a 401/403 from upstream; the storefront shows “Service is temporarily unavailable”.

  1. Open Admin → Settings → API management and confirm the key is set.
  2. Re-copy the key from the ChangeNOW partner dashboard (no leading or trailing whitespace).
  3. Check Admin → API logs — recent failed calls log the raw upstream response.
  4. If the partner account was just created, allow 5–10 minutes for the key to activate.

To verify the key against the live API from the command line:

curl -H "x-changenow-api-key: YOUR-KEY" \
  https://api.changenow.io/v2/exchange/currencies

Queue worker not running

If swaps stay waiting forever and the admin Horizon page is empty:

  1. Confirm Horizon is running: ps aux | grep horizon.
  2. If not, start it manually: php artisan horizon. For production, supervise it with systemd or supervisord.
  3. Check storage/logs/horizon.log for fatal startup errors.
  4. If Redis is in use, confirm it is reachable: redis-cli ping should return PONG.
  5. Confirm the cron entry exists and runs every minute: crontab -l.
Tip. After deploying a code change, run php artisan horizon:terminate so a fresh worker picks up the new code.

Mail not sending

  • Run the SMTP test: php artisan tinker → Mail::raw('test', fn ($m) => $m->to('you@you.com')->subject('hi'));
  • Inspect storage/logs/laravel.log for transport errors.
  • Verify your MAIL_FROM_ADDRESS uses a domain that passes SPF/DKIM — most providers reject anonymous senders.
  • For Gmail / Google Workspace, use an App Password, not your real password.

Migrations failed

If the installer's database step fails partway through, or you see “SQLSTATE… relation already exists”:

  1. Confirm the DB user has CREATE, ALTER, and DROP privileges.
  2. Drop the database and recreate it empty: 
    # MySQL
mysql -u root -p -e "DROP DATABASE crossswap; CREATE DATABASE crossswap CHARACTER SET utf8mb4;"

# PostgreSQL
sudo -u postgres psql -c "DROP DATABASE crossswap; CREATE DATABASE crossswap;"

# SQLite
rm database/database.sqlite && touch database/database.sqlite
  3. Re-run from the CLI to see the full error: 
    php artisan migrate --force
  4. If a migration mentions a missing PHP extension (e.g. bcmath), install it via apt or your hosting panel.

Permissions errors on storage/ and bootstrap/cache/

Symptoms: “The stream or file could not be opened”, blank white screen, or 500 from the very first request.

  1. Make sure the web server user (e.g. www-data, nginx, or apache) owns the writable directories: 
    sudo chown -R www-data:www-data storage bootstrap/cache
  2. Set sane permissions: 
    find storage bootstrap/cache -type d -exec chmod 775 {} \;
find storage bootstrap/cache -type f -exec chmod 664 {} \;
  3. On shared hosting where you cannot change ownership, use 777 as a last resort.
  4. Clear caches once permissions are right: 
    php artisan cache:clear
php artisan config:clear
php artisan view:clear

HTTPS mixed-content warnings

If the browser console shows “Mixed Content: The page was loaded over HTTPS, but requested an insecure resource”:

  1. Set APP_URL=https://yourdomain.tld in .env.
  2. Force the scheme in App\Providers\AppServiceProvider::boot(): 
    if ($this->app->environment('production')) {
    \\Illuminate\\Support\\Facades\\URL::forceScheme('https');
}
  3. If you're behind Cloudflare or a load balancer, configure the TrustProxies middleware so Laravel honours the X-Forwarded-Proto header.
  4. Clear caches: php artisan config:clear && php artisan view:clear.

Installer redirect loop

If the site keeps redirecting to /install after you completed the wizard, the lock file probably wasn't written. Create it manually:

touch storage/app/installed.lock
chmod 644 storage/app/installed.lock

If the opposite is true and you can't reach the installer because the lock file pre-exists on a fresh install, move it aside:

mv storage/app/installed.lock storage/app/installed.lock.bak

Admin login fails

  • Double-check the admin email in the admins table.
  • Reset the password with a fresh bcrypt hash: 
    php artisan tinker --execute="\
  \\App\\Models\\Admin::where('email','you@example.com')->first()\
  ->update(['password' => bcrypt('new-password-12chars')]);"
  • If 2FA is enabled and you've lost the device, clear the two_factor_secret column to disable it.
  • If the issue is a CSRF token mismatch, clear the session storage: php artisan session:flush (or redis-cli flushdb if you store sessions in Redis).

White screen / 500 error after upload

  1. Check the permissions on storage/ and bootstrap/cache/ (see above).
  2. Look at storage/logs/laravel.log — the exception will be there.
  3. If the log is empty, check your web server's error log (/var/log/nginx/error.log or /var/log/apache2/error.log).
  4. Make sure APP_KEY is set in .env — if blank, the app fails before logging. Re-generate with php artisan key:generate.
